Building the landing page

The formal term for the main page of a website is "landing page". Landing pages are often the first thing someone sees when visiting the website, so we want to make sure it's well made. Again, the two prompts will focus on building a prototype landing page for UXLA.

Before you begin

Before you dive into coding, there are few things to note:

Make sure you are in the dev branch when you code, add, commit, and push.

To move into the dev branch, open your terminal at the UXLA directory and run the follwoing command:

/UXLA/$ git checkout dev

Beware of conflicts

This prompt is rather unusual in that many people will be working on the same file at the same time. To make sure you don't accidentally overwrite other people's work, or vice versa, run git pull frequently on your computer to ensure you have the latest version of the code. Don't forget to communicate and coordinate with other members to update each other on what you are doing.

Ask questions!
